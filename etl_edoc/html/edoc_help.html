How This API Document Is Organized

This API (Application Programming Interface) document has pages corresponding to the items in the navigation bar, described as follows.

Overview

The Overview page is the front page of this API document and provides a list of all packages, as well as user defined types, with a summary for each.

Package

Each e verification package has a page that contains a list of its structs, units and sequences, with a summary for each.

Struct/Unit/Sequence

Each struct, Unit and Sequence struct has its own separate page. Each of these pages has two sections consisting of a description and member descriptions tables:

  • Struct Like inheritance diagram
  • Direct When Substructs
  • Struct description

  • Field Table
  • Method Table
  • Event Table
  • Cover Group Table

Tree (Struct Like Hierarchy)

There is a Struct Like Hierarchy page for all erm packages, plus a hierarchy for each erm package. Each hierarchy page contains a list of structs. The structs are organized by like inheritance structure starting with any_struct
  • When viewing the Overview page, clicking on "Tree" displays the hierarchy for all erm packages.
  • When viewing a particular erm package or struct, clicking "Tree" displays the hierarchy for only that package.

Index

The Index contains an alphabetic list of all erm packages, user types, structs and struct members.
